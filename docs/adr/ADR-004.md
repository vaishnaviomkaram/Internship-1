# ADR-004: Using LLM-as-Judge for RAG Evaluation

**Status:** Accepted  
**Date:** 2026-07-22  
**Deciders:** Vaishnavi  

---

## Context

A RAG system needs evaluation beyond “the answer looks good.” We need a repeatable way to judge whether answers are grounded in the retrieved context, whether citations are accurate, and whether the answer is complete.

Manual evaluation is reliable but slow. Traditional NLP metrics such as BLEU or ROUGE are not ideal because RAG answers can be phrased in many valid ways. We needed a practical evaluation method for a 5-week internship MVP.

---

## Decision

We will use an LLM-as-Judge evaluation module.

The evaluator receives:

- the question
- the retrieved context
- the generated answer

It scores the answer on three axes:

1. Correctness
2. Citation precision
3. Completeness

Each axis is scored from 1 to 5.

The evaluator prompt explicitly instructs the model to assign a low correctness score if the answer contains outside knowledge or hallucinations.

---

## Consequences

| Aspect | Positive | Negative / Risk |
|---|---|---|
| Speed | Faster than fully manual grading. | Still requires human review for high-stakes evaluation. |
| Cost | Uses the same LLM provider; no extra tooling required. | Adds API usage cost/quota consumption. |
| Consistency | Provides a structured scoring format. | LLM judges can be biased or inconsistent. |
| Debugging | Feedback field explains weak answers. | Feedback may itself be imperfect. |
| MVP fit | Good enough for internship-scale evaluation. | Production system should add golden datasets and automated metrics. |

---

## Alternatives Considered

| Alternative | Reason for Rejection / Delay |
|---|---|
| Manual-only evaluation | Too slow for iterative prompt changes. |
| BLEU / ROUGE | Poor fit for open-ended answer quality. |
| Ragas / DeepEval | Strong options, but added extra setup time during a 5-week MVP. Planned for 3rd-year extension. |
| Human expert panel | Not feasible within internship timeline. |

---

## Conclusion

LLM-as-Judge is a practical MVP evaluation method. It allows rapid feedback on correctness, citation quality, and completeness. In 3rd year, this should be extended with automated batch evaluation, golden answer datasets, and retrieval metrics.
